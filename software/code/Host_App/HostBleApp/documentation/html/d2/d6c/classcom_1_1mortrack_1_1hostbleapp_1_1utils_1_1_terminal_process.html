void com.mortrack.hostbleapp.utils.TerminalProcess.runCommand ( String  command,
int  timeout,
boolean  verbose 
) throws Exception

Executes a desired Terminal Command.

Aside from executing a desired Terminal Command, this method also prints on the Stream some informational notifications (such as the Directory Path Location at which this Terminal Command was executed and the actual Terminal Command requested to be sent) and the received response from sending that Terminal Command.

Parameters
commandThe Terminal Command that wants to be executed.
Note
The implementer is responsible for giving the right value to the command param according to the Operative System from which the com.mortrack.hostbleapp.utils.TerminalProcess Class is being used on.
Parameters
timeoutTimeout in seconds that the implementer wishes this to run at the most.
verboseWhenever set to true , it will show on the Stream some informational notifications about the Terminal Command requested to be executed and of the response received after running it. Otherwise, if its set to false , then it will not show anything on the Stream.
Note
By Stream, it is meant that java.lang.System.out.println will be used to send messages.
Exceptions
java.lang.Exception

Runs a desired Executable file with respect to the root location of this project and gets the first response line given after having executed that file.

Parameters
commandThe Execute Terminal Command of the File that wants to be executed.
Note
The implementer is responsible for giving the right value to the command param according to the Operative System from which the com.mortrack.hostbleapp.utils.TerminalProcess Class is being used on.
An example valid value for the command param in Windows is
command = ".\\APIs\\main.o 1 2 3"
and for Linux based O.S. is
command = "./APIs/main.o 1 2 3"
.
Parameters
timeoutTimeout in seconds that the implementer wishes this to run at the most.
Returns
Provides the first response line given after having executed the file indicated via the command param. However, if there in the case that there is not response at all, then the "" String value will be given instead.
