Detailed Description

Firmware Update Configurations Blocks parameters structure. This contains all the fields needed to write/read/erase the Firmware Update Configurations Blocks.

Note
If the size of this struct is to be changed, then make sure that the bytes size of the designated pages of the Firmware Update Configurations sub-module (which is 4096 bytes) are divisible by the resulting size of firmware_update_config_t .
For more details on the total size of all the designated Flash Memory pages of the Firmware Update Configurations sub-module , see the FW_UPDT_CONFIG_PAGE_SIZE definition and have in mind that the Firmware Update Configurations sub-module states that this sub-module has been designated 4 Flash Memory pages.
The aligned(4) attribute is used here because this struct must be a multiple of 4 bytes (i.e., 32-bits), since that is the configured TypeProgram with which our MCU/MPU's Flash Memory was configured with (see write_data_to_flash_app and FLASH_TYPEPROGRAM_WORD ). However, it is best if you set all the fields of this firmware_update_config_t struct to types of 4-bytes instead of having the aligned(4) attribute for filling the required size of this struct. The reason for this is because that attribute will not initialize all those additional bytes to zeros or ones, and will set them to random values instead, which means that this could change the CRC32 calculation each time you write a new data block (note that as long as the CRC for each data block is correctly calculated, then that should not affect the functionality of this sub-module, but will have more difficult data to follow/read).
