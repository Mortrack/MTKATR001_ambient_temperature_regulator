Detailed Description

ETX OTA Status Monitor parameters structure. This contains all the fields of the data that will be required so that the implementer/user of the Firmware Update module can both determine if an ETX OTA Transaction has been completed or not, and what was its resulting ETX OTA Status Exception Code.

Note
The way in which this parameter structure is expected to be used by the user/implementer of the Firmware Update module is to create a local variable of this ETX_OTA_Status_monitor_t and to update its values via the update_status_monitor function.

ETSM_is_read_flag ETX_OTA_Status_monitor_t::has_ec_been_read_before

Flag that indicates whether the latest value updated in the exception_code field has been read or not.

Note
This field can also be used by the implementer to tell if an ETX OTA Transaction has been concluded whenever its value is ETSM_is_read_flag::ETSM_HAS_NOT_BEEN_READ . On the other hand, in the case that its value is ETSM_is_read_flag::ETSM_HAS_BEEN_READ instead, then that will mean that no ETX OTA Transactions have recently been completed.
